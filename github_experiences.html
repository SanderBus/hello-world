10 mei 2016 -- First experiences with git and github

Github glossary

Branch - Making a copy of the repository (which still lives in that repository), such that one can make changes without affecting the original, which lives in the master branch. You can merge the copy bakc into the master branch when you are happy with the changes.
Clone - Copy of a repository on your local machine.
Commit - Saves the file(s) you worked on, including a log message. Each commit gets a unique ID(hash or SHA), such that older versions of the code can be recovered.
Push - Act of pushing an updated file from your local machine to github.
Repository - Project folder, this is what github is about. It not only saves files with code, but also documentation and revision history. Multiple people can work on one project.
Source: Github glossary

How to connect git and github

  • Clone project from github to local directory:
    git clone https://sanderbus@github.com/SanderBus/project.git
  • You can now locally alter the files as desired.
  • Once you are happy with the progress you can commit to it (+give appropriate comments for future devolopers:)
    git add program.py
                        git commit -m "Commit message"
  • If your done you can push the files back to github:
    git push origin master
    Or to another branch then master
